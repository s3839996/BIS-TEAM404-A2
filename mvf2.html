MVF2 - Quick Bookings

Description

Quick bookings feature will allow the user to enjoy a quick 2 click booking for parking. This will consist of the most frequent bookings on the home page. Once a user selects their preferences, all they need to do is click pay to automatically complete transaction using a credit card or other payment method that is already on file. The second part of the quick booking scenario includes the ability to use the phones native voice control system to book a parking spot. Allow the user to mention to the voice control system the place of booking and the duration. Time of start does not need to be mentioned since the time limit will begin once the user enters.


Lucid Textual Explanation

  • Upon tapping on 'Book Parking,' the user will be prompted to pick a destination using the map within the application to determine where they intend to go. The user will also have a list of frequent locations to select from if they have been known to go to certain places regularly. This will be in a small menu below the application.

  • The user will then enter a date and time as to which they'll need the parking space. The user will also have to select which vehicle they are using from the list below.

  • The application will then give a list of the closest free parking spaces based on the location the user wishes to go to. This will be a scroll-able menu, with also a choice of payment method needed at the bottom to know where it will be charged.

  • Finally, the user will be given a summary of the booking to double check that all details are correct. Once they are, they may choose to either go back and change details, or accept have the transaction processed to their card.


User Stories

"As a manager, I want to save my time so that I don't arrive late to work"

The quick booking feature should allow the user to swiftly and effortlessly make bookings within a given period. The users should find it convenient to navigate through and also be able to manage their time efficiently. The Quick Park application should be able to prompt users to select, confirm and finalise the parking lot and have it ready for the user on the day and time that they require. Ultimately, the app will help users save time in searching for parking and can help them get to work stress-free and effortlessly. Individuals will feel a bit of relief that they have a back-up solution when it comes to parking and will be more prepared when it comes to going somewhere.


"As a beginner driver, I want to have a secured route with parking so I can avoid getting stressed"

A new driver will be less assertive in their parking skills and may require either more time to develop confidence. The Quick Park will be able to offer quick guidance in registering and be able to allow the user to not only find parking conveniently within proximity of their location of choice and secure it for the user. This will able to help the novice drivers to build confidence as they have parking secured in a location that they are comfortable and are not required to worry about parking, especially in busy locations. Having the application handy, will allow them to reduce stress and continue to build their confidence in both driving and parking in new unfamiliar locations.


"As a mum of three, I want to find a convenient parking location, so I can drop my children off to kindergarten safely and effortlessly"

For individuals who have children or undertake a lot of responsibility, the Quick Park application can offer the guidance for them to find parking and be able to carry out their daily tasks safely and smoothly without any additional tension. The app will allow the user to browse through and select the nearest convenient available parking lot which then can be booked and secured. This way the user can carry out through their daily tasks and possibly reduce their stresses when it comes to parking.

Validation Criteria

There are several validation types that can be used to check data and information that is being entered by a user.

  • Format check: this is used to check if data is in the correct format. In relation to the quick bookings page, this can be used when the user enters a date and time for their parking reservation, as these are examples of data fields that need to be entered in a specific format. The date and time could also be validated by utilising a date / time picker on the device.

  • Existence check: this is used to check if data has been entered at all. In relation to the quick bookings page, all fields such as location, date, time, vehicle, parking space and payment method will need to be validated as having something entered in the field.

  • Lookup table: this is used to check for acceptable values in a table. In relation to the quick bookings page, when the user specifies the parking address, it needs to be cross checked using an external database such as google maps, in order to confirm that the address exists.

We also need to determine a list of acceptance criteria to help us set a user's expectation for our application. This can be done by generating possible scenarios that a user may experience and outline the expectations of that scenario. We can generate this using the Given/When/Then template.

    Scenario: User makes a booking

    "Given I'm in a role of a registered user

    When I open the bookings page


    Then the system shows me location form containing "Location of Choice" and "Frequent Locations" of which only one of these is required

    When I fill in "Location of Choice" with a valid location

    Or When I select a "Frequent Location" from the provided list

    And I click the "Continue" button


    Then the system shows me the booking form containing "Date", "Time" and "Choose a Vehicle" which are required fields

    When I fill in the "Date" with a valid date

    And I fill in the "Time" with a valid time

    And I "Choose a Vehicle" by using the checkboxes

    And I click the "Next" button


    Then the system shows me the booking form containing "Select Available Parking Space" and "Payment" which are required fields

    And I "Select Available Parking Space" by using the checkboxes

    And I select a "Payment Method" by using the checkboxes

    And I click the "Next" button


    Then the system shows me the booking summary screen

    And I click the "Confirm Booking" button


    Then the system shows me the booking confirmed screen"
