USER STORIES

  • Researching about the type of users that would like likely be using the application.
  • Talking to potential clients and analysing their needs and desires of the ideal application.
  • Analysing how the current application and MVF's will be able to cater the needs of the user.
  • Designing the layout of the user stories and aligning it to the existing features.

ARCHITECTURE DIAGRAM

  • Researching about architecture diagrams and how to construct one.
  • Analysing the types of information that is required to be added into the diagram.
  • Provide justification and explanation of the architectural design and how it is beneficial for the user.

VALIDATION CRITERIA

  • Research about the structure of the validation criteria.
  • Compare the MVF's to what is expected by the user and how the expectations will be met.

LUCID TEXT DESCRIPTION OF QUICK BOOKING MINIMUM VIABLE FEATURE

  • Explore the feasibility of the feature.
  • Research about competitor applications and their use of the feature.
  • Detailed written paragraph about the feature and what the user can expect when using the application.

SET UP AND CREATION OF WEBSITE

  • Commitments to GitHub.
  • Creation of a design template on Visual Studio.
  • Checking and correcting completed work (refining stage).
