MVF4 - Payment

Description

The payment section will consist of a quick and easy solution to calculate ticket fares. The app will utilize the internet time and location to determine what the time is for the user's current location, which is then used to determine the fare for the user (whether its peak time or not). If the user is calculating a fare for a future booking, they can enter the location and time manually to get the corresponding fees/prices.

The app will consist of several cashless payment solutions. This includes an online terminal credit card processor, PayPal and device specific services such as (Google pay, Apple Pay, Samsung pay). In some cases, paying cash at the payment window/terminal will be available depending on the location. At the end of transaction, the user will be reminded that they can save their payment details for any future transactions. This would come with the ability to print a receipt at the end of a certain transaction, which can then be saved to the users native file system or sent as an email.


Lucid Textual Information

  • Upon tapping on 'Manage Payments,' the user will be menu with their payment methods displayed, in which they may choose to remove a payment method if needed. If this is their first time entering payment information, there will only be given a button that says 'Add Payment Method,' which will also be present if an existing user chooses to add more payment methods.
  • A variety of payment options will be supported, such as credit card, PayPal, etc. These will be select-able when booking a parking space, and can be saved within the application.
  • Each payment method will be represented by an icon of sorts, either of the card being used or an icon of the service being used (e.g, PayPal).


User Stories

"As a technologically incompetent person, I would like to have a quick and easy payment method"

Quick Park is working on developing a quick and smooth payment method, where the user can enter the payment details and make secured bookings just within a few taps. The user is only required to add the card details once and it will be stored on file, so it is convenient, and the user does not need to enter the same information multiple times. The card details which are placed on file can then later be edited in the case that card details have changed. The application will also prompt first-time users on how to set up the payment option to have an easy experience when using the application.


"As an individual who is tight on budget, I would like to see the transactions that have been made through the application"

Quick Park will indicate to the user a list of the times and the parking locations that the users have booked in the past and will indicate a summary of the savings and total monthly cost that has been used.
